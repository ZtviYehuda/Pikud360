## Component Standard Document Structure

Every component specification page strictly follows the 9-point structure required by Enterprise UI Sprint 35:

1. **Purpose** — Core user purpose & usage criteria
2. **Props** — Complete TypeScript props signature & descriptions
3. **Examples** — Copy-pasteable React usage code
4. **Variants** — Visual representations & variant combinations
5. **Accessibility** — Keyboard support & ARIA compliance
6. **RTL** — RTL layout behavior & mirroring rules
7. **Theme** — Light/Dark mode token mapping
8. **Responsive** — Mobile, Tablet & Desktop adaptive behavior
9. **Best Practices** — Specific DOs and DON'Ts

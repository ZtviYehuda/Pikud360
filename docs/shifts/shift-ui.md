# Shift and Scheduling UI Architecture

**Domain:** Shift Management / Scheduling  
**Phase:** 14.6 — Shift UI Architecture  
**Depends on:** shift-data-contracts.md, scheduling-rules.md, shift-api.md

---

## 1. Overview

This document specifies the UI Architecture for the Shift and Scheduling module. It outlines layout hierarchies, page contexts, custom sub-components, routing structures, and data flow mechanisms.

---

## 2. Pages

---

### 2.1 Shift Calendar (לוח שנה חודשי)

**Route:** `/scheduling/calendar`  
**Purpose:** Monthly grid overview of scheduled shifts, displaying coverage levels at a glance.

#### Responsibilities
- Renders the month overview as a grid of `CalendarCell` components.
- Displays summary indices per day (total slots, coverage percentages).
- Provides filters (search, unit hierarchy selectors).
- Clicking any date cell opens the `DailySchedule` view or navigates directly to that day's workspace.

#### Layout Structure
- **Header Toolbar**: Month switcher, unit filters, legend indicator, and "תכנון שבועי" (Weekly Planner) navigation shortcut.
- **Calendar Grid**: Standard 7-column month grid (Sunday-Saturday).
  - Each cell is a `CalendarCell` displaying coverage percentage.

---

### 2.2 Daily Schedule (לוח זמנים יומי)

**Route:** `/scheduling/day/:date`  
**Purpose:** Focused list view of shifts scheduled for a specific date.

#### Responsibilities
- Loads the date's shifts and assignments via `ShiftCalendarDTO` slice.
- Renders a list of `ShiftCard` panels.
- Highlights empty slots, showing "+ הקצה" (Assign) action triggers.
- Displays constraint warnings directly on slot rows (e.g. "התרעת מנוחה חסרה").

#### Layout Structure
- **Day Header**: Date title (e.g. "יום שני, 20 ביולי"), navigation arrows (previous/next date), and `CoverageIndicator` summary bar.
- **Shifts List**: Vertical stack of `ShiftCard` panels (Morning, Evening, Night).

---

### 2.3 Weekly Planner (מתכנן שבועי)

**Route:** `/scheduling/planner`  
**Purpose:** A horizontal Gantt-like weekly board where unit commanders plan and populate shifts.

#### Responsibilities
- Renders a 7-day grid canvas (columns = days, rows = shift types).
- Displays drag-and-drop slots or click-to-assign grids.
- Highlights unfilled slots dynamically using red warning borders.
- Surfaces a floating **Warnings Panel** listing all active constraint violations (overtime, rest buffer overlaps) across the weekly roster.
- Renders the "Publish Schedule" workflow action.

#### Layout Structure
- **Planner Canvas**: A grid showing days as columns and shift cards as rows.
- **Warnings Sidebar**: Real-time listing of conflicts generated by the validation API.

---

### 2.4 Assignment Dialog (חלון שיבוץ)

**Component Scope:** Dialog / Pop-up Form  
**Purpose:** The form popup to assign an employee to a shift slot.

#### Responsibilities
- Loads eligible personnel list from `GET /employees` filtering by the slot's required qualifications.
- Selecting an employee triggers validation rules against existing schedules.
- If a **Hard Block** constraint is violated:
  - Disables the save button and renders a red alert card (e.g. "חריגה ממגבלת 12 שעות יומית").
- If a **Soft Block** constraint is violated:
  - Renders a yellow alert card and shows a mandatory text field for `overrideReason`.
- Submits assignments to `POST /shifts/{id}/assign`.

#### Layout Structure
- **Roster Selection**: Searchable list of unit employees, showing their active statuses.
- **Alert Banner**: Dynamic validation warnings.
- **Override Section**: Renders `overrideReason` text input when soft blocks are detected.

---

### 2.5 Coverage View (מדדי כיסוי)

**Route:** `/scheduling/coverage`  
**Purpose:** Reporting view summarizing unit shift coverage statistics.

#### Responsibilities
- Renders graphical gauges of scheduling coverage (filled vs. unfilled ratios) over a selected date range.
- Details overtime analytics (personnel exceeding standard weekly limits).
- Lists critical operational gaps (roles unstaffed, e.g. "מחסור בנהגים מוסמכים").

## 3. Shared Components

---

### 3.1 Shift Card (`SchedulingShiftCard`)

**Purpose:**  
Renders a visual card block representing a shift instance.

**Responsibilities:**
- Renders the shift name, times, duration, and owner unit.
- Displays a vertical list of requirement slots (e.g., Commander, Guard, Driver).
- For filled slots, renders the `AssignmentBadge`.
- For unfilled slots, renders a dotted "+ הקצה" placeholder.
- Color highlights: Amber left border for shifts with empty slots; Green border for fully staffed shifts.

---

### 3.2 Calendar Cell (`SchedulingCalendarCell`)

**Purpose:**  
A single date cell grid block in the monthly view.

**Responsibilities:**
- Renders the date day number.
- Displays the daily `CoverageIndicator` (color-coded circular gauge).
- Renders icon indicators for active warnings (e.g., caution triangle if a soft block was bypassed).
- Inactive days (outside the current month) are rendered with greyed-out text and empty states.

---

### 3.3 Assignment Badge (`SchedulingAssignmentBadge`)

**Purpose:**  
Renders an assigned employee's details inside a shift slot.

**Responsibilities:**
- Displays avatar, rank, and full name.
- Renders a click-to-edit action trigger.
- Renders an icon warning next to the name if the assignment carried a soft block override, hovering opens a tooltip detailing the override reason.
- Renders a distinct visual border if the employee is assigned in a standby/on-call state instead of physical presence.

---

### 3.4 Coverage Indicator (`SchedulingCoverageIndicator`)

**Purpose:**  
Renders coverage ratios and percentages.

**Responsibilities:**
- Circular progress ring or linear progress bar showing filled slots vs. required slots.
- Color codes:
  - Green (≥ 95% coverage)
  - Yellow (75%–94% coverage)
  - Red (< 75% coverage)
- Renders subtitle text (e.g. "8/9 איישו").

## 5. State Ownership & Data Flow

```
          API (GET /shifts/calendar)
                       │
                       ▼
            useShiftCalendar (Custom Hook)
                       │
                       ▼
          WeeklyPlannerPage (owns selectedWeekState)
                       │
                       ├─────► Renders Grid canvas
                       │         └──► SchedulingShiftCard
                       │                 └──► SchedulingAssignmentBadge
                       │
                       ├─────► Renders WarningsSidebar
                       │
                       └─────► User clicks unfilled slot
                                   │
                                   ▼
                            AssignmentDialog
                                   │ (validates selections)
                                   ├──► calls constraint check API
                                   │       └──► returns validation warnings
                                   │
                                   ▼ (submits selection DTO)
                            POST /shifts/{id}/assign
```

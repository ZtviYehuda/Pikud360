# Report Export Specifications

**Domain:** Reporting  
**Phase:** 17.4 — Report Exports  
**Depends on:** reporting-domain.md, report-catalog.md, report-filters.md

---

## 1. Overview

This document specifies the format structures, metadata headers, naming conventions, and authorization limits applied to downloadable files (PDF, Excel, CSV) exported by the Pikud360 reporting engine.

---

## 2. Supported Formats Reference

---

### 2.1 PDF (דוח מודפס)

- **Layout:**
  - Standard A4 portrait or landscape layout (configurable per template).
  - RTL (Right-to-Left) Hebrew rendering with grid borders and page page-breaks.
  - Page margins set to 15mm.
  - Font families restricted to pre-compiled Hebrew web-safe fonts (e.g. Outfit, Heebo, David Libre).
  - Contains structural charts (e.g., bar charts for strength stats) and official signature blocks at the footer.
- **Included Metadata:**
  - **Header Block**: Tenant Logo, Report Title, Generating Unit Name (e.g., מדור א').
  - **Footer Block**: Page indicator (`Page X of Y`), Generator metadata (`Generated by User Name (Rank) at YYYY-MM-DD HH:MM:SS UTC`), Classification label (`מקור מוגבל - לשימוש פנימי בלבד`).
  - **Filters Card**: Summary table detailing applied parameters (applied date range, active node scopes).
- **File Naming Convention:**
  - Pattern: `[ReportCode]_[ScopeCode]_[DateUTC].[Extension]`
  - Example: `WFSUM_MDR-A-7_2026-07-19.pdf`
- **Permissions:**
  - `reporting.export.pdf` permission at the target scope.
  - Access is locked to Section Heads (Level 4) and above for workforce files.

---

### 2.2 Excel (גיליון נתונים מעובד)

- **Layout:**
  - Styled spreadsheet (.xlsx format).
  - Contains two sheets:
    - **גיליון סיכום (Summary Sheet)**: Styled dashboard sheet with aggregate cards and formula sums.
    - **נתונים (Data Sheet)**: Flat rows with auto-width columns and formatted data cells.
  - Uses the tenant's brand palette (Slate grey headers, white cell backgrounds).
  - Number fields formatted as integers; dates formatted as local dates (`YYYY-MM-DD`).
- **Included Metadata:**
  - **Summary Sheet Header**: Row 1-4 contains the report title, author metadata, extraction timestamp, and query parameters summary table.
  - **Author Properties**: Excel document properties (Author, Created By) are set to "Pikud360 Reporting Engine".
- **File Naming Convention:**
  - Pattern: `[ReportCode]_[ScopeCode]_[DateUTC].[Extension]`
  - Example: `ATTEND_GD-51_2026-07-19.xlsx`
- **Permissions:**
  - `reporting.export.excel` permission at the target scope.
  - Access restricted to commanders and admin auditors to prevent large roster downloads by general operators.

---

### 2.3 CSV (קובץ נתוני פסיק)

- **Layout:**
  - Plain text comma-separated values (.csv format).
  - Uses `,` (comma) as the column separator and `\n` as the newline indicator.
  - Encoding must be **UTF-8 with BOM (Byte Order Mark)**.
    - *Reason:* Enforces Hebrew character compatibility when users open CSV files directly in Microsoft Excel (preventing gibberish text rendering).
  - No colors, formulas, or cell formatting.
- **Included Metadata:**
  - **No inline headers**. To keep the file clean for data imports, metadata is excluded from the text body.
  - Basic file properties (creation date, scope) are embedded in the file name.
- **File Naming Convention:**
  - Pattern: `[ReportCode]_[ScopeCode]_[DateUTC].[Extension]`
  - Example: `SHIFTS_GD-51_2026-07-19.csv`
- **Permissions:**
  - `reporting.export.csv` permission at the target scope.
  - Allowed for automation integrations (System Administration webhook flows).

## 3. Format Capabilities Matrix

This matrix details which export formats are supported by each report in the catalog.

| Report | PDF | Excel | CSV | Notes |
|---|---|---|---|---|
| **Workforce Summary** | Yes | Yes | Yes | PII fields masked in all formats. |
| **Attendance Report** | Yes | Yes | Yes | CSV contains raw check-in timestamps. |
| **Shift Coverage** | Yes | Yes | No | PDF contains visual coverage bars. |
| **Organization Report**| Yes | No | Yes | CSV used for tree import schemas. |
| **Employee Status** | Yes | Yes | Yes | PDF outlines sickness/leave categories. |
| **Birthday Report** | Yes | No | No | Print-only social calendar template. |
| **Certification** | Yes | Yes | Yes | Excel contains expiry formulas. |
| **Audit Report** | Yes | Yes | Yes | CSV used for security ingestion tools. |

# מערכת כניסה מהירה מאובטחת עם PIN

## סקירה כללית

מערכת כניסה מהירה מאובטחת המבוססת על קוד PIN שמוצפן באמצעות SHA-256. המערכת מספקת חוויית משתמש דומה לכניסה ביומטרית, אך עובדת בכל סביבה (כולל Cloudflare Tunnel) ומספקת אבטחה טובה יותר מאשר כניסה אוטומטית ללא אימות.

## רכיבים

### 1. PinVerificationModal
**קובץ:** `frontend/src/components/auth/PinVerificationModal.tsx`

מודל מעוצב לאימות קוד PIN:
- 6 שדות קלט נפרדים (4-6 ספרות)
- מעבר אוטומטי בין שדות
- תמיכה בהדבקה (paste)
- אימות אוטומטי כשמוזנות 6 ספרות
- הודעות שגיאה ברורות
- אנימציות חלקות

### 2. ProfileSettings - הגדרת PIN
**קובץ:** `frontend/src/components/settings/ProfileSettings.tsx`

תהליך הגדרה דו-שלבי:
1. **אימות סיסמה:** המשתמש מזין את הסיסמה הנוכחית שלו
2. **יצירת PIN:** המשתמש יוצר קוד PIN בן 4-6 ספרות ומאשר אותו

**אבטחה:**
- הסיסמה מאומתת מול השרת לפני יצירת PIN
- ה-PIN מוצפן באמצעות SHA-256 עם salt (שם המשתמש)
- הסיסמה המקורית מאוחסנת מוצפנת ב-base64 (לשימוש בכניסה)

### 3. LoginPage - כניסה עם PIN
**קובץ:** `frontend/src/pages/LoginPage.tsx`

תהליך כניסה:
1. משתמש לוחץ על כפתור הכניסה המהירה (אייקון טביעת אצבע)
2. נפתח מודל אימות PIN
3. משתמש מזין את קוד ה-PIN שלו
4. המערכת מאמתת את ה-PIN מול ה-hash השמור
5. אם ה-PIN נכון, המערכת מבצעת login אוטומטי

## אבטחה

### הצפנת PIN
```typescript
const hashPin = async (pin: string, username: string): Promise<string> => {
  const encoder = new TextEncoder();
  const data = encoder.encode(pin + username); // Salt with username
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
};
```

### אחסון ב-localStorage
- `biometric_pin_${username}` - PIN hash (SHA-256)
- `biometric_token_${username}` - פרטי התחברות מוצפנים (base64)
- `biometric_registered_${username}` - דגל רישום
- `biometric_last_user` - שם המשתמש האחרון

### יתרונות האבטחה
1. ✅ **אימות משתמש:** דורש הזנת PIN לפני כניסה
2. ✅ **הצפנה:** PIN מוצפן עם SHA-256 + salt
3. ✅ **אימות סיסמה:** הסיסמה מאומתת מול השרת לפני יצירת PIN
4. ✅ **עובד בכל סביבה:** לא דורש WebAuthn או HTTPS מיוחד
5. ✅ **חוויית משתמש:** דומה לכניסה ביומטרית אמיתית

### מגבלות
- ⚠️ אחסון ב-localStorage (לא מוצפן ברמת הדפדפן)
- ⚠️ לא משתמש בחומרת ביומטריה אמיתית של המכשיר
- ⚠️ הסיסמה מאוחסנת מקומית (מוצפנת ב-base64)

## שימוש

### הפעלת כניסה מהירה
1. עבור להגדרות → פרופיל
2. גלול לסעיף "כניסה מהירה עם PIN"
3. הזן את הסיסמה הנוכחית שלך
4. לחץ "המשך להגדרת PIN"
5. צור קוד PIN בן 4-6 ספרות
6. אשר את הקוד
7. לחץ "צור PIN"

### כניסה עם PIN
1. בדף ההתחברות, לחץ על כפתור טביעת האצבע
2. הזן את קוד ה-PIN שלך במודל
3. המערכת תכניס אותך אוטומטית

### ביטול כניסה מהירה
1. עבור להגדרות → פרופיל
2. לחץ "בטל כניסה מהירה"
3. כל הנתונים השמורים יימחקו

## השוואה ל-WebAuthn

| תכונה | PIN System | WebAuthn |
|-------|-----------|----------|
| עובד ב-Cloudflare Tunnel | ✅ כן | ❌ לא |
| דורש HTTPS יציב | ❌ לא | ✅ כן |
| ביומטריה אמיתית | ❌ לא | ✅ כן |
| תמיכה בדפדפנים | ✅ כל הדפדפנים | ⚠️ חלקי |
| אבטחה | ⭐⭐⭐ טובה | ⭐⭐⭐⭐⭐ מצוינת |
| חוויית משתמש | ⭐⭐⭐⭐ טובה מאוד | ⭐⭐⭐⭐⭐ מצוינת |

## המלצות לעתיד

כאשר המערכת תעלה לאוויר עם דומיין יציב:
1. שקול מעבר ל-WebAuthn לביומטריה אמיתית
2. הוסף הצפנה ברמת הדפדפן (Web Crypto API)
3. שקול שימוש ב-IndexedDB במקום localStorage
4. הוסף timeout אוטומטי למחיקת נתונים שמורים

## תיעוד טכני

### Flow Diagram

```
[משתמש] → [לחיצה על כפתור PIN] → [PinVerificationModal]
                                              ↓
                                    [הזנת 6 ספרות]
                                              ↓
                                    [Hash PIN + Username]
                                              ↓
                                    [השוואה ל-Stored Hash]
                                              ↓
                                    [אם תואם: Login אוטומטי]
                                              ↓
                                    [ניווט לדף הבית]
```

### API Calls
- `POST /api/auth/login` - אימות סיסמה בעת יצירת PIN
- `POST /api/auth/login` - כניסה אוטומטית עם פרטים שמורים
